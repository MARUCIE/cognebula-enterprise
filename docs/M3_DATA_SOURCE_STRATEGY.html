Executive Summary

执行摘要

CogNebula 知识图谱在 M2 阶段完成了 Ontology v2.2 迁移，节点规模达到 408K，边数 860K，密度 2.109，质量门分数 100。但距离 M3 里程碑目标（1M 节点 / 6M 边 / 密度 6.0）仍有显著缺口。本次蜂群审计对全部 22 个爬虫的采集深度、18 个遗漏高价值信源、以及图谱密度增长的系统动力学进行了全面审查。

45% 的爬虫仅抓取标题，未采集正文全文 -- 这是图谱密度停滞的首要瓶颈。

审计发现三个关键问题：第一，22 个爬虫中仅 4 个达到 SOTA 深度（全文 + 结构化提取），10 个停留在浅层采集（仅标题/摘要），3 个已损坏或未启用；第二，18 个高价值信源尚未接入，其中 8 个建议立即启动；第三，边增长必须达到节点增长的 7 倍才能实现密度目标 -- 这意味着策略必须从"广度优先"转向"深度优先"。

密度悖论

当前 860K 边 / 408K 节点 = 密度 2.109。要在 1M 节点时达到密度 6.0，需要 6M 边。缺口为 +5.14M 边。新增节点如果不配套大规模边生成，密度反而会下降。深度（边）优先于广度（节点）。

Density Strategy

密度战略

图谱密度是衡量知识连通性的核心指标。当前密度 2.109 意味着每个节点平均仅有 2.1 条边，远低于目标值 6.0。密度提升不能靠简单堆砌节点 -- 必须在节点增长的同时，以 7 倍速率增长边数。

策略核心：深度优先于广度。每新增 1 个节点，必须配套生成 6+ 条边。

四大边增长引擎：

  1. chinatax_api 全文提取 -- 170K 节点升级为全文后，自动提取法规引用链（REFERENCES / AMENDS / SUPERSEDES），预估生成 170K+ 边
  2. LR QA 生成管线 -- 基于全文生成问答对，每个 QA 节点关联原文 + 知识点，预估 120K 节点 + 120K 边（管线已运行中）
  3. 边富化定时任务 -- 关键词匹配生成 RELATES_TO / SIMILAR_TO 边，覆盖跨域关联（税法-会计准则、税法-海关编码等）
  4. AI 推理边 -- 利用 Gemini 推理生成 SUPERSEDES / CONFLICTS_WITH / IMPLEMENTS 时序链，覆盖法规演进关系
