2. 用户旅程图

旅程 A：AI 智能体税务查询

触发
用户向 OpenClaw 提问
"软件企业增值税优惠政策有哪些？"
步骤 1
OpenClaw 路由至 CogNebula MCP 工具
调用 tax_query 工具
步骤 2
CogNebula 混合 RAG 检索
  • LanceDB 语义搜索 -- 返回 top-5 相关文档
  • KuzuDB 图谱遍历 -- 政策 -> 税种 -> 行业链
  • 分层上下文组装（4K token 预算）
步骤 3
返回结构化 Markdown
  • 适用政策（含生效日期）
  • 税率 + 免税条件
  • 相关条款（全文）
  • 判例先例（如有）
步骤 4
OpenClaw 综合生成回答
将结构化上下文整合为自然语言回复
结果
用户收到有据可查、附带引用的税务指导

质量门禁

  • Precision@5 >= 85%（前 5 条结果中相关结果占比）
  • 响应时间 < 3 秒（MCP 往返）
  • Token 预算遵守：上下文 <= 4K tokens

旅程 B：每日法规监控

触发
VPS 定时任务启动
每日 06:00 UTC，finance-tax-crawl.sh 7 阶段流水线开始执行
步骤 1
AI-Fleet 技能抓取器爬取 5 个活跃数据源
顺序执行，每源间隔 5 秒礼貌延迟
  • chinatax.gov.cn：132 条（非 SPA 回退方案）
  • mof.gov.cn：50 条（正则解析器）
  • pbc.gov.cn：10 条（条法司）
  • safe.gov.cn：60 条（外汇管理局，3 个板块）
  • csrc.gov.cn：177 条（证监会，3 个板块）
步骤 2
变更检测（按数据源）
  • 哈希比对：是否存在二进制变更？
  • 如有变更：unified_diff 识别变更段落
  • 如为重大变更：embedding 相似度 < 0.95
步骤 3
新增/变更内容处理
  • Trafilatura 文本提取
  • 基于规则的 NER（法规编号、日期、税率）
  • KuzuDB 节点 upsert + 边关系创建
  • LanceDB 向量嵌入生成
步骤 4
告警推送
Telegram 推送："NOTE: 检测到 3 条新政策"
  • 政策 A：[标题]（生效日期：YYYY-MM-DD）
  • 政策 B：[标题]（修订：[旧政策编号]）
  • 政策 C：[标题]（行业：软件）
结果
CF Pages 仪表盘更新
  • 时间线视图：本周所有变更
  • 影响分析：受影响的行业/税种
  • 全文 + 差异高亮

质量门禁

  • 爬取完成率：全部 15+ 数据源均已检查
  • 变更检测延迟：发布后 24 小时内
  • 告警送达：检测后 30 分钟内
  • 一级数据源变更零漏检

旅程 C：合规检查工作流

触发
用户提问
"我们公司是杭州的软件企业，今年要缴哪些税？"
步骤 1
OpenClaw 意图解析
  • 管辖区域：杭州市（浙江省）
  • 行业：软件（GB 65）
  • 纳税人类型：企业（默认一般纳税人）
  • 时间：2026 年度
步骤 2
MCP 工具调用：compliance_check
输入：{industry: "软件", jurisdiction: "杭州", taxpayer: "一般纳税人", year: 2026}
步骤 3
图谱遍历
  • 管辖区(杭州) -> 上级 -> 管辖区(浙江) -> 上级 -> 管辖区(国家)
  • 各层级：GOVERNS -> 法规
  • 筛选：status=active, effective_date <= 2026-03-14
  • 行业(软件)：APPLIES_TO <- 税种(*)
  • 纳税人类型(一般纳税人)：SCOPED_TO <- 政策(*)
步骤 4
组装合规清单
  • 增值税：6%（软件服务）/ 13%（软件产品），优惠：即征即退（超 3% 部分）
  • 企业所得税：25%（标准）/ 15%（高新技术）
  • 附加税：城建税 + 教育费附加
  • 印花税：合同类
  • 地方性优惠：浙江省特有政策
结果
返回带引用的合规清单
[法规编号] + [条款号] + [有效期]

4. 异常状态与恢复

异常 用户影响 恢复策略
爬虫被政府网站拦截 数据过期（无更新） 更换 UA 重试；连续 3 次失败后告警
KuzuDB 查询超时 MCP 响应变慢 缓存高频查询；10 秒超时返回部分结果
Embedding API 不可用 语义搜索失效 降级为 KuzuDB 文本搜索（query_symbols）
PDF OCR 失败 文档内容缺失 加入人工审核队列；图谱中标记 ocr_failed
变更检测误报 不必要的告警 告警中附带置信度分数；阈值设为 0.95
