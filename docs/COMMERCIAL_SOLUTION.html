CogNebula Enterprise

企业级智能体上下文大脑 (Context Brain for AI Agents)

完全本地化、纯净闭源友好的 SOTA 级代码知识图谱解决方案，赋予大模型精准的架构级“物理直觉”。

一、商业化解决方案概述

核心痛点与定位

在企业级 AI Agent（如智能研发助手、自动化测试 Agent、安全审计 Agent）落地的过程中，单纯依赖大模型的向量检索（Vector RAG）往往会遭遇严重的“上下文截断”与“幻觉”问题。大模型懂语义，但不懂物理调用链路与爆炸半径。

CogNebula Enterprise 的定位是：作为中立的、高保真的“大脑后端”，将企业庞大的多语言代码库抽象为精准的拓扑知识图谱。通过原生提供 Graph RAG 接口，向所有的企业智能体无缝投喂“高密度的防膨胀上下文”。

SOTA 级核心功能特性

架构创新

Late-Binding 混合检索 (Hybrid Graph-Vector RAG)

底层融合了 LanceDB (向量语义) 与 KuzuDB (结构图谱)。智能体不仅能通过“自然语言”找到代码，还能瞬间穿透图谱提取上下游 3 层深度的强相关调用链。

成本优化

自适应上下文降级 (Tiered Detail Degradation)

为防止撑爆 LLM 的 Token 窗口，引擎支持按图谱深度智能剪裁：核心命中节点返回完整代码 (Depth 0)，调用方仅返回函数签名 (Depth 1)，远端模块仅返回文件路径 (Depth 2)。

企业安全

多租户物理隔离 (Shared-Nothing Isolation)

摒弃 Neo4j 等巨石数据库。基于 KuzuDB 嵌入式引擎，为每个代码仓库/租户建立独立的磁盘级物理数据库，彻底杜绝企业间“数据串库”和并发写锁问题。

极致体验

影视级 3D WebGL 可视化控制台

内置 3d-force-graph 引擎，支持同屏 10 万+节点 60fps 流畅渲染。管理人员可像浏览星系一样俯瞰企业资产架构，并支持节点点击触发镜头飞行与下钻分析。

数据时效

毫秒级事件驱动入库 (Event-Driven Worker)

内置 Redis 队列支持。支持对接 GitLab/GitHub Webhook，企业代码一旦 Merge，后台守护进程即刻增量更新图谱，保障 Agent 推理的上下文永远新鲜。

100% 容错

全量 Tree-sitter AST 解析

不仅支持标准 Python/JS/TS 的多进程正则提权，SOTA 版本更原生集成 Tree-sitter，即使企业的代码未编译或存在严重语法错误，依然能构建出完整拓扑图谱。

二、一键部署指南

环境前置要求

  • 操作系统: 任意支持 Docker 的 Linux (Ubuntu/CentOS), macOS, 或 Windows WSL2。
  • 硬件要求: 最低 4 核 CPU，8GB 内存 (KuzuDB 的流式落盘机制对内存极为宽容)。
  • 网络要求: 纯本地/私有云支持，可 100% 断网离线运行，无任何外网 License 验证或数据回传。

部署步骤 (Docker Compose 方式)

CogNebula Enterprise 被封装为一个高内聚的容器编排栈（API Gateway + Worker Daemon + Redis）。

Step 1: 获取商业交付包与导入数据

# 1. 克隆企业版仓库或解压交付包
git clone git@github.com:YourCompany/cognebula-enterprise.git
cd cognebula-enterprise

# 2. 将需要分析的企业代码库挂载到 data 目录下
mkdir -p data/my-java-backend
# (将企业的业务代码复制到此处)

Step 2: 一键拉起微服务集群

# 后台编译并启动 SOTA 容器群
docker-compose up -d --build

# 查看运行状态
docker-compose ps

Step 3: 访问与验证

  • 可视化控制面板: 打开浏览器访问 http://127.0.0.1:8766，即可沉浸式浏览生成的 3D 代码图谱。
  • API 接口文档 (OpenAPI/Swagger): 访问 http://127.0.0.1:8766/docs，获取标准的 RESTful 接口定义。

三、智能体 (Agent) 接入指南

部署完成后，您的企业内部智能体编排框架（如 LangChain, Dify, FastGPT 等）可通过统一的 Graph RAG 端点接入大脑。

Graph RAG 黄金接口

POST /api/rag

请求体 (Payload):

{
  "repo": "/data/my-java-backend",
  "query": "请告诉我用户鉴权模块的核心逻辑，以及如果我修改它会影响哪些下游 API？"
}

返回体 (Response): 该接口会自动执行【向量检索命中】->【Cypher 图遍历】->【上下游 3 层扩展】->【Token 降级裁剪】，并返回一段开箱即用的高密度 Markdown 上下文给大模型。
