# 场景：发票真伪验证流程

**政策依据**：《发票管理办法》第三十三条；国家税务总局全国增值税发票查验平台

## 第一步：验证渠道选择

| 渠道 | 适用范围 | 操作方式 |
|------|----------|----------|
| 全国增值税发票查验平台 | 所有增值税发票 | https://inv-veri.chinatax.gov.cn |
| 电子发票服务平台 | 全电发票 | 登录税务数字账户查询 |
| 各省电子税务局 | 本省发票 | 登录后在发票查询模块 |
| 微信/支付宝 | 快捷查验 | 搜索"发票查验"小程序 |

**决策规则**：首选全国增值税发票查验平台（覆盖最全），全电发票直接在电子发票服务平台验证状态。

## 第二步：查验操作步骤

**纸质/电子增值税发票**：
1. 输入发票代码（12位）
2. 输入发票号码（8位）
3. 输入开票日期
4. 输入校验码后6位（普票）或金额（专票，不含税金额）
5. 验证码识别后查验

**全电发票**：
1. 输入20位发票号码
2. 输入开票日期
3. 输入税价合计金额
4. 查验

**查验结果解读**：
| 结果 | 含义 | 处理 |
|------|------|------|
| 查验一致 | 发票真实有效 | 正常入账 |
| 查无此票 | 发票可能为假 | 拒绝接收，要求对方换开 |
| 不一致 | 发票信息被篡改 | 退回核实 |
| 已作废 | 发票已被作废 | 退回要求重开 |
| 已红冲 | 发票已被红字冲销 | 退回核实 |

## 第三步：批量验证方案

**单张查验限制**：平台每日查验次数有限（通常每张票查验5次/天）。

**批量方案**：
1. 财务系统内置OCR识别+API查验（需对接税务数据接口）
2. 第三方发票管理平台（如票易通、百望等）批量查验
3. 税务数字账户中直接查看已接收的全电发票

## 第四步：异常处理与风险控制

- **假发票处理**：发现假发票应立即拒绝报销，通知开票方。如已入账需冲回并补开正规发票
- **失控发票**：已认证抵扣的发票被列为失控/异常的，税务机关通知后需做进项转出（可申诉恢复）
- **查验时效**：建议在收到发票3日内完成查验（注意：2020年起增值税专用发票认证期限已取消，不再有360天限制，但及时查验仍是最佳实践）
- **建立查验制度**：所有进项发票入账前必须通过查验平台验真，金额超过1万元的必须查验
- **电子发票查重**：电子发票可重复打印，必须建立发票号码台账防止同一张发票重复报销
