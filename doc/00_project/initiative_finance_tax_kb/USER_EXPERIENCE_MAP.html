2. 用户旅程地图

旅程 A: AI 智能体税务查询

触发: 用户向 OpenClaw 提问「软件企业增值税优惠政策有哪些？」
▼
1OpenClaw 路由至 CogNebula MCP 工具: tax_query
▼
2CogNebula 混合 RAG 检索
  • LanceDB 语义搜索 -- 返回 Top-5 相关文档
  • KuzuDB 图谱遍历 -- 政策 → 税种 → 行业链
  • 分层上下文组装（4K Token 预算）
▼
3返回结构化 Markdown
  • 适用政策（含生效日期）
  • 税率 + 减免条件
  • 相关条款（全文）
  • 判例参考（如有）
▼
4OpenClaw 综合生成回答
▼
用户获得: 有据可查、引用明确的税务指导

质量门禁

  • Precision@5 ≥ 85%（Top 5 结果相关率）
  • 响应时间 < 3 秒（MCP 往返）
  • Token 预算遵守: 上下文 ≤ 4K Tokens

旅程 B: 每日法规监控

06:00 UTC -- Celery Beat 触发爬取管线
▼
1Scrapy 爬虫并行爬取 15+ 数据源
  • chinatax.gov.cn: 最新公告
  • mof.gov.cn: 政策发布
  • gov.cn: 国务院决定
▼
2变更检测（逐源）
  • 哈希比对: 是否有二进制级变更？
  • 如有变更: unified_diff 识别变更段落
  • 如为重大变更: embedding 相似度 < 0.95
▼
3新增/变更内容处理
  • Trafilatura 正文提取
  • 基于规则的 NER（法规编号、日期、税率）
  • KuzuDB 节点更新 + 关系创建
  • LanceDB 向量生成
▼
4告警推送
  • Telegram: 「NOTE: 检测到 3 条新政策」
  • 政策 A: [标题]（生效日: YYYY-MM-DD）
  • 政策 B: [标题]（修订自: [旧政策]）
  • 政策 C: [标题]（行业: 软件）
▼
5CF Pages 仪表板更新
  • 时间线视图: 本周所有变更
  • 影响分析: 受影响的行业/税种
  • 全文附差异高亮

质量门禁

  • 爬取完成率: 全部 15+ 数据源已检查
  • 变更检测延迟: 发布后 < 24 小时
  • 告警送达: 检测后 30 分钟内
  • Tier-1 数据源变更零漏报

旅程 C: 合规检查工作流

触发: 用户提问「我们公司是杭州的软件企业，今年要缴哪些税？」
▼
1OpenClaw 解析意图
  • 辖区: 杭州市（浙江省）
  • 行业: 软件（GB 65）
  • 纳税人类型: 企业（默认一般纳税人）
  • 时间: 2026 年度
▼
2MCP 工具: compliance_check
  • 输入: {industry: "软件", jurisdiction: "杭州", taxpayer: "一般纳税人", year: 2026}
▼
3图谱遍历
  • Jurisdiction(杭州) → parent → Jurisdiction(浙江) → parent → Jurisdiction(国家)
  • 每层级: GOVERNS → 法规列表
  • 过滤: status=active, effective_date ≤ 2026-03-14
  • Industry(软件): APPLIES_TO ← TaxType(*)
  • TaxpayerType(一般纳税人): SCOPED_TO ← Policy(*)
▼
4组装合规清单
  • 增值税: 6%（软件服务）/ 13%（软件产品）-- 优惠: 即征即退（超 3% 部分）
  • 企业所得税: 25%（标准）/ 15%（高新技术）
  • 附加税: 城建税 + 教育费附加
  • 印花税: 合同类
  • 地方专属: 浙江省地方优惠
▼
返回结果附引用: [法规编号] + [条款号] + [有效期]

旅程 D: 专家简报生产

计划: 每日 09:30 CST（复用现有专家团管线）
▼

▼
3交付管线（复用现有）
  • HTML 海报（800x1120, Bloomberg 风格）
  • Playwright 截图
  • Telegram 推送（截图 + 3 行摘要）
  • CF Pages: ai-fleet-dashboard.pages.dev/finance-tax-digest-YYYY-MM-DD.html

4. 异常状态与恢复

异常 用户影响 恢复策略
爬虫被政府网站封禁 数据过期（无更新） 更换 UA 重试；连续 3 次失败后告警
KuzuDB 查询超时 MCP 响应缓慢 缓存高频查询；10 秒超时返回部分结果
Embedding API 不可用 语义搜索失效 降级为 KuzuDB 文本搜索（query_symbols）
PDF OCR 失败 文档内容缺失 进入人工审核队列；图谱中标记 ocr_failed
变更检测误报 不必要的告警 告警附置信度分数；阈值设定为 0.95
