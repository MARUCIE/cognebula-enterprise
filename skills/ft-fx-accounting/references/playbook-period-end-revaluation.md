# 场景：期末外币重估

**政策依据**：《企业会计准则第19号——外币折算》第十二条、第十三条

## 第一步：确定需要重估的外币货币性项目

按准则第十二条，资产负债表日应对外币货币性项目采用资产负债表日即期汇率折算。货币性项目包括：
- **资产类**：银行存款（外币户）、应收账款、应收票据、其他应收款、外币债券投资
- **负债类**：短期/长期借款（外币）、应付账款、应付票据、其他应付款

非货币性项目（存货、固定资产、预付/预收账款、实收资本等）不进行期末重估，仍按交易日汇率保持不变。

## 第二步：计算汇兑差额

汇兑差额 = 资产负债表日外币余额 × 期末汇率 - 该项目当前账面人民币余额。

示例：应收账款USD 100,000，入账时汇率7.10（账面710,000），期末汇率7.25：
- 重估后金额 = 100,000 × 7.25 = 725,000
- 汇兑收益 = 725,000 - 710,000 = 15,000

分录：
- 借：应收账款 15,000
- 贷：财务费用—汇兑损益 15,000

对于外币负债（如应付账款），汇率上升意味着需要支付更多人民币，产生汇兑损失。

## 第三步：批量重估操作流程

月末/季末/年末重估步骤：
1. 导出所有外币货币性项目余额表（按币种汇总）
2. 获取资产负债表日央行外汇中间价
3. 逐户/逐币种计算汇兑差额
4. 编制汇兑损益汇总表，经财务主管复核
5. 生成记账凭证，过入总账

ERP系统通常有自动外币重估功能，但应手工验证：抽取3-5个科目核对系统重估结果是否正确。特别注意：有些系统默认用月末最后一天汇率而非央行中间价，需确认一致。

## 第四步：披露与税务处理

财务报表附注中应披露：外币货币性项目的外币金额和折算汇率、当期确认的汇兑损益金额。期末汇兑损益属于未实现损益，税务处理参见fx-loss-tax-deduction playbook。月度重估差异在次月初不自动冲回（区别于某些ERP默认设置），而是在后续交易结算或下次重估时自然调整。

关键输出：外币重估计算表 + 汇兑损益汇总 + 重估凭证 + 非货币性项目排除清单。
