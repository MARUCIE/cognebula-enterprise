# 场景：RAG问答模式模板

**政策依据**：RAG（Retrieval-Augmented Generation）技术应用于财税知识问答

## 第一步：问题分类

| 问题类型 | 特征 | 检索策略 |
|----------|------|----------|
| 事实查询 | "XX的税率是多少" | 精确匹配→直接返回 |
| 条件判断 | "我这种情况能不能享受优惠" | 条件提取→规则匹配 |
| 操作指导 | "怎么办理XX手续" | 流程检索→步骤输出 |
| 比较分析 | "A方案和B方案哪个税负低" | 多路检索→对比计算 |
| 风险评估 | "这样做有什么税务风险" | 场景匹配→风险库检索 |

## 第二步：问题改写模板

将用户口语化问题改写为结构化检索查询：

**原始问题**："我们公司买了一台50万的设备，能不能一次扣掉？"

**改写为**：
```
实体: 固定资产/设备
金额条件: 500万以下
查询目标: 一次性税前扣除政策
检索关键词: "设备器具 一次性扣除 500万 企业所得税"
关联法规: 财税〔2018〕54号
```

**其他改写示例**：
| 口语问题 | 结构化查询 |
|----------|-----------|
| "招待费能报多少" | 业务招待费 + 税前扣除 + 限额 + 60% + 5‰ |
| "我们要不要登记一般纳税人" | 一般纳税人 + 登记条件 + 500万 + 年应税销售额 |
| "发票丢了怎么办" | 发票丢失 + 处理流程 + 2020年第1号公告 |
| "给员工发福利要交啥税" | 职工福利 + 个人所得税 + 增值税 + 进项转出 |

## 第三步：回答生成规则

**结构化回答模板**：
```
[结论]：直接给出答案（是/否/金额/期限等）
[依据]：引用具体法规条款（文号+条款号）
[条件]：列出适用条件（满足哪些才能适用）
[注意]：特殊情况或例外
[操作]：具体如何操作（如需）
```

**回答质量标准**：
- 每个回答必须至少引用一个法规依据
- 引用的法规必须经过时效性验证
- 涉及金额/比例/期限的必须给出精确数字
- 存在多种情况的必须分情况说明

## 第四步：高频问答对维护

建立并持续更新高频问答库：
- 统计用户高频提问TOP 50
- 为每个高频问题预构建标准答案
- 标准答案每季度更新一次（政策变动时立即更新）
- 新问题模式被识别后及时补充到问答库

**维护规则**：
- 答案中的法规引用过期 → 立即更新
- 用户反馈答案不准确 → 72小时内修正
- 同类问题出现新变体 → 补充到改写模板库
