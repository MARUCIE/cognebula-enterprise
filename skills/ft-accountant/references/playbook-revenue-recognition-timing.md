# 场景：收入确认时点判定

**政策依据**：《企业会计准则第14号——收入》（财会〔2017〕22号）；《企业会计准则第14号——收入》应用指南（2018）

## 第一步：判别履约义务类型

区分在某一时段内履行还是在某一时点履行的履约义务：
- **时段确认**（满足以下任一即可）：(1) 客户在企业履约的同时即取得并消耗经济利益（如保洁服务）；(2) 企业的履约过程创造的资产客户能够控制（如在客户土地上建造）；(3) 资产具有不可替代用途且企业有权收取已完成部分的款项（如定制化软件）
- **时点确认**：不满足上述任何一项的，按时点确认

## 第二步：时点确认的五项标志

对于时点履行的履约义务，判断控制权是否转移（需综合考虑）：
1. 企业就该商品享有现时收款权利
2. 客户已拥有该商品的法定所有权
3. 企业已将该商品实物转移给客户
4. 客户已取得该商品所有权上的主要风险和报酬
5. 客户已接受该商品（验收）

**决策规则**：五项标志不要求全部满足，但第4项（风险报酬转移）和第5项（客户验收）在实务中是最关键的判定依据。发出商品但客户未验收的，不得确认收入。

## 第三步：常见易错场景核查

| 场景 | 正确时点 | 常见错误 |
|------|----------|----------|
| 附有退货条款 | 发出时按预期退货率估计，确认净额收入 | 全额确认，退货时冲减 |
| 委托代销 | 收到代销清单时 | 发出商品时 |
| 预收款销售 | 发出商品时 | 收到预收款时 |
| 分期收款 | 合同约定收款日 | 商品发出时全额确认 |
| 附安装调试 | 安装检验完毕时 | 发货时 |

## 第四步：账务处理与风险提示

确认收入时借记"应收账款/合同资产"，贷记"主营业务收入""应交税费——应交增值税（销项税额）"，同时结转成本。

**风险提示**：收入确认时点错误直接导致企业所得税季度预缴偏差和增值税纳税义务时间错位，被税务稽查调整的概率极高。尤其注意增值税纳税义务发生时间（《增值税暂行条例》第十九条）与会计收入确认时点可能不一致。

**关联技能协同**：
- 收入确认时点与增值税纳税义务时间不一致时 --> `ft-tax-accounting-gap` 处理税会差异调整
- 涉及合同条款判定（含退货/分期/安装条款） --> `ft-business-analyst` 审查合同涉税要点
- 跨期收入确认影响所得税 --> `ft-tax-advisor` 评估季度预缴和汇算清缴影响
- 收入确认异常可能触发稽查预警 --> `ft-compliance-auditor` 进行收入完整性自查
