关键架构决策

Decision 1

Shared-Nothing 隔离

每仓库 / 每租户独立 KuzuDB 目录，硬件级隔离。

Decision 2

晚期绑定 Hybrid RAG

LanceDB 找语义入口，KuzuDB 映射爆炸半径（blast radius）。

Decision 3

Tree-sitter 通用解析

带错误恢复的通用 parser，替代 JS/TS 的 regex 路径。

Decision 4

事件驱动摄入

Webhook → Redis → 单线程 ingester（per-repo），保证写入顺序。

Decision 5

API Gateway 前置

JWT 校验 + 租户路由先于任何 DB 访问。

§ 5 · 前端表面拓扑（跨项目契约 · 2026-04-28 边界更正）

Two-Frontend Topology（CogNebula 持有 1 个）

对称写约束：canonical 编辑权属于 30-lingque-agent/doc/00_project/initiative_lingque_agent/SYSTEM_ARCHITECTURE.md §1.3 Two-Frontend Topology。本节将 CogNebula Enterprise 在该拓扑里的角色固定写入，防止两仓库静默漂移。

公司平台对外暴露恰好两个前端。CogNebula Enterprise（System A）持有其中一个：

显式 Out-of-Scope（2026-04-28 边界更正）

Boundary correction

下列两类不属于本架构图的覆盖范围，不要混入：

  • wiki.hegui.cn 是内部研发文档管理系统，不是面向客户或面向公司平台的表面；归内部工具范畴，不在公司产品拓扑中。
  • yiclaw.hegui.io 与 25-yiclaw 仓是 Maurice 个人项目，不是公司项目，也不是"架于 System B 之上的 applied product"。有独立部署与所有权，必须保持在公司架构文档之外。

KG 不可见性原则

Hard rule · KG Invisibility

本范围内唯一的客户层表面是 hegui.app 非 operator 视图——绝对不能暴露指向 hegui.io 的人类可点击链接。KG 由 agent 通过 MCP server / CF Worker 代理消费，不由人类导航。审计义务在 30-lingque-agent；CogNebula 的义务是通过 Cloudflare Access 拒绝非 agent UA（advisory，直至 enforcement middleware 上线）。yiclaw 已不在公司表面之列，本原则范围由"两个客户层"收窄至单一客户层。

迁移 Backlog 对本仓库的影响（M1 退役后）

  • M5（审计 hegui.io 是否被 hegui.app 非 operator 视图入链）— 唯一直接触及本仓库的 backlog 项；触发时走一次 referer-allowlist 扫描，范围已收窄至单一客户层
  • M2（跨注册根 SSO 桥）/ M3（?return_to= + sso_token 解析）/ M4（header 链接 + operator 层 gating）不在本仓库管辖
  • M1（yiclaw 子域分裂）已退役—— yiclaw 不在公司范围内，故此项消失
