# 场景：审计计划编制流程

**政策依据**：《中国注册会计师审计准则第1201号——计划审计工作》；《企业内部控制审计指引》第十条

## 第一步：了解被审计单位及其环境

收集被审计单位基本信息：行业特征、监管环境、组织架构、会计政策选择、近期重大交易。重点关注上年度审计发现的未整改事项。按照CAS 1211，执行风险评估程序，识别重大错报风险领域。

关键输出：被审计单位概况表 + 初步风险评估矩阵。

## 第二步：确定重要性水平与审计范围

依据CAS 1221《计划和执行审计工作时的重要性》，确定整体重要性水平（通常为税前利润的5%或营业收入的0.5%-1%）。设定实际执行的重要性水平（通常为整体重要性的50%-75%）。明确审计范围：全面审计还是专项审计，哪些科目需要重点审计。

关键输出：重要性水平计算表 + 审计范围确认书。

## 第三步：编制审计计划与资源配置

制定总体审计策略和具体审计计划。内容包括：审计目标、审计期间、审计团队成员及分工、时间进度安排、需要利用的专家工作（CAS 1421）。评估是否需要利用内部审计工作（CAS 1411）。确定关键审计程序和抽样方法。

关键输出：审计计划书（含人员分工表 + 时间甘特图 + 程序清单）。

## 第四步：计划沟通与动态调整

将审计计划与被审计单位管理层沟通，确认配合事项和资料提供清单。根据CAS 1201第十二条，审计计划在审计过程中应当根据情况变化及时修订。建立定期沟通机制，记录计划变更及原因。

关键输出：审计通知书 + 资料提供清单 + 计划变更日志。

**风险提示**：
- 重要性水平设定过高可能遗漏重大错报，设定过低则审计成本过高
- 审计计划未及时修订可能导致审计程序与实际风险脱节
- 未充分了解被审计单位环境即开展实质性程序，违反审计准则要求
