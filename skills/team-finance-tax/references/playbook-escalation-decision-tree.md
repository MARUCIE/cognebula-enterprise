# 场景：升级决策树——何时转人工

**政策依据**：《注册会计师法》第二十一条（执业标准与职业判断）；《代理记账管理办法》（财政部令第98号）第十五条（重大事项报告义务）

## 第一步：定义升级触发条件

AI专家系统应自动处理的范围：法规查询、标准计算、流程指引、模板生成、历史案例检索。以下情况必须升级到人工专家：

**L1 信息不足升级**：
- 用户提供的信息不足以做出判断（如"这笔交易怎么交税"但未说明交易性质/金额/主体）
- 需要获取企业内部非公开信息（如具体合同条款、内部审批记录）
- 涉及的事实尚在发展中（如税务稽查进行中，结论未出）

**L2 专业判断升级**：
- 法规模糊地带：条文有多种解读且无权威案例
- 金额重大：单笔影响超过100万元的税务处理决策
- 跨辖区差异：涉及多地税务机关，各地执行口径不一致
- 税务筹划方案：涉及交易结构设计的主动税务规划

## 第二步：升级决策树

```
用户提出问题
│
├─ 是否为标准法规查询/计算？
│   YES → AI自动处理 → 输出结果
│   NO ↓
│
├─ 信息是否完整？
│   NO → 向用户追问必要信息 → 补齐后重新判断
│   YES ↓
│
├─ 是否涉及金额>100万的决策？
│   YES → L2升级（标注：金额重大，建议人工复核）
│   NO ↓
│
├─ 法规是否有明确规定？
│   YES → AI处理 + 标注置信度
│   NO ↓
│
├─ 是否有权威案例/先例？
│   YES → AI参考案例处理 + 标注风险提示
│   NO → L2升级（标注：法规模糊，需专业判断）
│
├─ 是否涉及税务筹划/交易设计？
│   YES → L2升级（标注：需要人工设计方案）
│
├─ 是否涉及诉讼/争议/稽查应对？
│   YES → L3升级（标注：必须人工主导）
```

## 第三步：升级信息传递模板

升级到人工时，AI应提供完整的上下文信息：

```
[升级请求]
用户问题：[原始问题]
已完成的分析：[AI已做的判断和计算]
升级原因：[L1信息不足 / L2专业判断 / L3诉讼争议]
具体不确定点：[需要人工判断的具体问题]
相关法规：[已检索到的相关法条]
初步建议：[AI的初步倾向性意见，标注置信度]
建议的人工专家类型：[税务师/律师/审计师/行业专家]
```

原则：升级不是"放弃"，而是"带着半成品转交"。AI应尽可能完成能做的部分，将人工专家的工作量降到最低。

## 第四步：升级后的闭环管理

人工专家处理完成后，结果应反馈到AI系统用于学习：
- 如果AI初步判断正确：增强该模式的置信度
- 如果AI判断错误：记录错误原因，更新知识库
- 如果属于新型问题：生成新的知识条目纳入知识图谱

长期目标：通过持续学习，逐步减少L2升级的比例，提高AI自主处理能力。但L3类问题（诉讼/稽查）始终保持人工主导。

关键输出：升级决策树（可视化）+ 升级触发条件清单 + 信息传递模板 + 闭环学习机制。
