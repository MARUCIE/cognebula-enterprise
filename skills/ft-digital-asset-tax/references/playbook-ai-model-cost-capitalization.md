# 场景三：AI模型训练成本资本化判定

**政策依据**：《企业会计准则第6号——无形资产》第7-9条（内部研发）；财会〔2023〕11号；财税〔2015〕119号（研发费加计扣除）

## 第一步：初步判别——AI模型是否可资本化

**AI模型训练的两个阶段**：
1. **研究阶段**：算法探索、数据可行性验证、技术路线评估 → **费用化**
2. **开发阶段**：确定模型架构后的训练、调优、部署 → **可资本化**（满足5个条件）

**资本化五个条件**（须全部满足）：
1. 完成该无形资产使其能使用或出售在技术上具有可行性
2. 具有完成该无形资产并使用或出售的意图
3. 该无形资产产生经济利益的方式（使用/许可/出售计划）
4. 有足够的技术/财务和其他资源支持完成开发
5. 归属于开发阶段的支出能够可靠计量

## 第二步：深入分析——AI模型成本构成

| 成本项目 | 金额通常占比 | 资本化判定 |
|---------|------------|----------|
| 算力/GPU费用 | 40-60% | 开发阶段→资本化；研究阶段→费用化 |
| 数据采集/标注 | 15-25% | 开发阶段→资本化；通用数据采集→费用化 |
| 研发人员工资 | 20-30% | 按工时分摊（研究/开发阶段比例） |
| 第三方服务费 | 5-10% | 跟随所服务的阶段 |
| 基础设施（云/存储） | 5-10% | 按使用比例分摊 |

**关键判定点**：何时从研究阶段转入开发阶段？
- 模型架构确定 + 数据集确定 + 基准性能验证通过 → 转入开发阶段
- 仍在尝试不同架构/数据集/训练策略 → 仍处于研究阶段

## 第三步：佐证/验证——研发加计扣除联动

**注意区分两个口径**：
1. **会计处理**：费用化→当期费用；资本化→无形资产
2. **研发加计扣除**：无论会计上费用化还是资本化，均可加计扣除100%

**资本化的AI模型加计扣除**：
- 资本化形成无形资产 → 按无形资产成本的200%在使用寿命内摊销扣除
- 公式：200%摊销 = 原始成本摊销 + 100%加计摊销

**实务建议**：
- 如果企业当期利润高、希望多扣除 → 倾向费用化（当期全额扣除+加计）
- 如果企业当期亏损 → 倾向资本化（延后到盈利期摊销+加计）

## 第四步：结论/建议

- **阶段划分文档化**：AI项目必须有书面的研究→开发阶段转换节点文档
- **工时记录**：研发人员参与AI模型训练的工时须按项目/阶段记录
- **算力成本追踪**：GPU/云服务费用须按项目分配，保留使用日志
- **加计扣除最大化**：即使资本化也可加计，确保辅助账归集完整
- **评估减值**：AI模型技术迭代快，定期评估是否存在减值迹象（如新模型已替代）

**风险提示**：
- 研究阶段与开发阶段划分缺乏书面文档，税务稽查时可能被要求全部费用化，丧失资本化收益
- AI模型使用寿命短（通常2-5年），资本化后短期内发生减值的概率较高
- 算力费用在多项目间分摊比例不合理，可能导致研发费加计扣除金额被调减

**关联技能协同**：
- 研发费加计扣除归集 --> `ft-tax-incentive` 确认研发费辅助账要求（playbook-rd-super-deduction-scope）
- 资本化后的摊销产生税会差异 --> `ft-tax-accounting-gap` 处理无形资产摊销差异
- 研发阶段费用确认 --> `ft-accountant` 遵循CAS 6无形资产准则
