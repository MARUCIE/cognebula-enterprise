# 场景：多专家协作流程模板

**政策依据**：《代理记账管理办法》（财政部令第98号）第十二条（专业胜任能力）；《中国注册会计师审计准则第1401号——利用其他注册会计师的工作》；《税务师职业道德规范》第八条

## 第一步：识别需要多专家协作的场景

单一专家无法独立完成的复杂问题需要多专家协作。典型场景：
- **跨税种问题**：如"企业重组的增值税、企业所得税、土增税、印花税分别怎么处理" → 需要4个税种专家协同
- **跨业务问题**：如"境外子公司的转让定价合规+外币报表折算+合并抵消" → ft-transfer-pricing + ft-fx-accounting + ft-consolidation
- **全链路问题**：如"新设子公司从注册到首次纳税申报全流程" → ft-business-registration + ft-vat-specialist + ft-corporate-tax + ft-period-calendar
- **审计+业务联动**：如"年度审计中发现的关联交易定价问题" → team-audit + ft-transfer-pricing

## 第二步：协作流程模板

**Phase 1: 需求分解（Router负责）**
- team-finance-tax接收用户问题
- 识别涉及的专家skill清单
- 确定协作模式：串行（有依赖关系）或并行（独立子问题）
- 生成任务分配表：每个专家负责的子问题 + 预期输出格式

**Phase 2: 独立执行（各专家负责）**
- 每个专家skill独立处理其负责的子问题
- 输出结构化结果：结论 + 依据（法条/准则）+ 计算过程（如有）+ 风险提示
- 标注与其他专家的衔接点（如"转让定价结论将影响合并抵消金额"）

**Phase 3: 整合与校验（Router负责）**
- 收集各专家输出
- 检查一致性：各专家的前提假设是否一致、数据口径是否统一
- 识别矛盾点：如税务优化建议与审计合规建议冲突
- 整合为统一答复

## 第三步：信息传递规范

专家间信息传递格式：
```
[FROM] ft-transfer-pricing
[TO] ft-consolidation
[CONTEXT] 关联交易定价调整后的影响
[DATA] 内部销售利润率从15%调整为12%，涉及金额500万
[ACTION_NEEDED] 请重新计算合并抵消中的未实现利润金额
```

数据一致性规则：
- 同一笔交易的金额在所有专家的分析中必须一致
- 税率和政策口径以Router确定的基准年度为准
- 企业基本信息（行业、规模、地区）由Router统一提供

## 第四步：交付物合并模板

最终交付物结构：
1. **问题概述**：用户原始问题 + 分解后的子问题列表
2. **各专家分析**：按子问题排列，每个子问题标注负责专家
3. **综合结论**：整合各专家意见的统一建议
4. **风险矩阵**：跨领域风险汇总（如税务风险、审计风险、合规风险交叉影响）
5. **行动清单**：按时间优先级排列的待办事项
6. **政策依据汇总**：所有引用的法条/准则/文件清单

关键输出：协作任务分配表 + 各专家结构化输出 + 一致性校验报告 + 综合分析报告。
