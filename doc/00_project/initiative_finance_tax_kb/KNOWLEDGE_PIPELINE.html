1. 决策日志

ADR-001: 管线拓扑 -- Obsidian 中心 vs NotebookLM 中心

背景: 需要一条 4 阶段管线: 爬取 → 消化 → 管理 → 发布。两个 AI 驱动系统可用: Google NotebookLM (anything-to-notebooklm 技能) 和本地 Obsidian 知识库。

#方案核心存储NotebookLM 角色风险复杂度
ANotebookLM 中心NotebookLM主消化 + 生成高 (认证过期, 速率限制, Google API 变更)中
BObsidian 中心Obsidian 知识库可选增强低 (完全本地)中
C双轨并行两者同时共同主体中 (同步冲突)高

决策: 方案 B -- Obsidian 中心

理由:

  1. notebooklm login 需要浏览器 OAuth; 无法在无人值守 cron 中运行
  2. NotebookLM 有未文档化的速率限制; 50+ 文件/天批量上传未经测试
  3. Obsidian 知识库 = Git 仓库 = 天然版本控制、差异比较和 CI/CD 集成
  4. NotebookLM 仍可按需使用 (手动或触发) 用于播客/FAQ 生成
  5. 单一真相原则: 知识库为权威存储; NotebookLM 为派生视图

3. 逐阶段设计

阶段 1 爬取 (已有)

脚本: scripts/finance-tax-crawl.sh | 输出: data/raw/{date}/*.json

使用技能: agent-reach, browser-automation, defuddle, multi-search-engine, news-aggregator, wechat-article-search, scrapling-crawler

信源: 20 个国内 (4 P0 政府 + 8 P1 专业 + 8 P2 媒体/社区)

无需变更。现有管线已达生产可用。

阶段 2 消化与入库 (双路输出)

阶段 2 从同一原始 JSON 输入分为两条并行路径:

路径 A: 图 + 向量 (已有)

  • finance_tax_processor.py: 基于规则的 NER, 3 级变更检测
  • KuzuDB: 13 节点中国税务本体 (18 税种, 42 CAS)
  • LanceDB: Gemini Embedding 2 向量, 通过 bin/embed

路径 B: Obsidian 转换 (新建)

新脚本: scripts/obsidian_converter.py (~200 行)

输入: data/raw/{date}/*.json + 处理器元数据 (NER 结果, 变更状态)

输出: 知识库中的 Obsidian 风格 Markdown 文件

Frontmatter 模式 (每文档):

---
source: "chinatax.gov.cn"
source_url: "https://fgk.chinatax.gov.cn/..."
crawl_date: "2026-03-15"
publish_date: "2026-03-14"
title: "关于调整增值税税率的公告"
regulation_number: "财税[2026]15号"
issuing_authority: "财政部 国家税务总局"
tax_type:
  - "增值税"
content_type: "regulation"  # regulation|guide|compliance|news|analysis
status: "active"            # active|expired|superseded|proposed
change_type: "new"          # new|updated|unchanged
tags:
  - "#法规"
  - "#增值税"
  - "#2026"
---

路径 C: NotebookLM (可选, 按需)

不在每日 cron 中。手动或通过 Claude Code 技能触发。

生成产物 (播客 MP3, FAQ Markdown, 思维导图 JSON) 保存到 _generated/ 目录。

5. NotebookLM 集成指南

前置条件

# 一次性设置 (需要浏览器 OAuth)
notebooklm login
notebooklm list  # 验证认证

批量上传知识库内容

# 上传本周所有法规
for f in ~/Obsidian/财税知识库/法规/**/*2026-03-1*.md; do
    echo "上传: $(basename "$f")"
    notebooklm source add "$f" --notebook "财税知识库" --wait
    sleep 3  # 速率限制保护
done

生成增强内容

# 每周播客摘要
notebooklm generate audio \
    --notebook "财税周报 2026-W11" \
    --instructions "中文总结本周重要财税政策变化, 面向企业财务人员, 5-8分钟" \
    --output ~/Obsidian/财税知识库/_generated/podcasts/2026-W11.mp3

# FAQ 文档
notebooklm generate report \
    --instructions "生成本周财税政策FAQ, 问答格式, 重点标注对中小企业的影响" \
    --output ~/Obsidian/财税知识库/_generated/faqs/2026-W11-faq.md

# 思维导图
notebooklm generate mind-map \
    --instructions "中国税种体系全景图, 18个税种的分类关系" \
    --output ~/Obsidian/财税知识库/_generated/mindmaps/tax-system-overview.json
