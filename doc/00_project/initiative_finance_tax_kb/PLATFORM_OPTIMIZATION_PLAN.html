2. 技术栈决策

组件选型理由备选方案
图数据库 KuzuDB (现有) 嵌入式、Cypher 原生、批量 CSV 加载、可离线 Neo4j (过重)、NebulaGraph (分布式过度)
向量数据库 LanceDB (现有) CogNebula 已集成，零配置 Milvus (需独立基础设施)、Qdrant (不必要)
嵌入模型 Gemini Embedding 2 Preview 免费、8K token、Matryoshka 维度、AI-Fleet 已集成 OpenAI (付费)、Jina (中文覆盖弱)
内容获取 AI-Fleet Skill 生态 40+ 生产级 Skill，零自建爬虫 自建 Scrapy (重复造轮子)
静态 HTML 抓取 agent-reach + defuddle 13+ 平台支持，干净提取 Scrapy (重复造轮子)
动态 JS 页面 browser-automation (Midscene) 视觉驱动，无需 DOM 选择器 Playwright 裸用 (代码量更大)
反爬绕过 scrapling-crawler 内置 Cloudflare Turnstile 绕过 自建中间件 (脆弱)
政策发现 multi-search-engine 17 个搜索引擎，零 API Key 单一搜索引擎 (覆盖有限)
深度分析 deep-research (GoT) 7 阶段管线、多智能体 人工综合 (缓慢)
内容提取 defuddle / x-reader 通用 URL 到干净文本 Trafilatura (未集成)
微信内容 wechat-article-search 基于镜像，比直接抓取更安全 直接微信 API (风险高)
新闻监控 news-aggregator-skill 内置 28 个来源，可扩展 自建 RSS 读取器 (多余)
PDF 提取 PyMuPDF + PaddleOCR PaddleOCR 中日韩识别最佳 Tesseract (中文识别差)
调度器 Fleet pipeline cron 62 条管线已验证，有降级链 Celery Beat (新增依赖)
RSS 生成 RSSHub (自托管) 政府网站缺少 RSS 源 手动轮询 (脆弱)
NER (第一阶段) 基于规则的正则 零训练数据需求 spaCy (需语料库)
