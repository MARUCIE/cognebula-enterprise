核心分拆原则：System A 是知识基础设施（内部使用），System B 是面向代账公司的产品。KG 的价值通过 Agent 准确率体现，而非让客户看图谱。

03

L2 — 数字员工（Agent Runtime）— 护城河

蜂群说"Agent 身份是仪式"。用户否决。在中国 B2B 代账市场，信任是个人化的 — 客户信任的是"林税安做了 6 个月零事故"，不是"Pipeline V3 执行成功"。信任记录不可迁移，这就是锁定。
01

信任锚点

每位 Agent 积累准确率、任务量、合规事故数、服务时长。客户切换竞品 = 丢失 6 个月的信任记录。

不可迁移 = 锁定

02

Hickey 简化

Agent 身份是纯配置（JSON），不是 700 行 TSX。能力映射到 Skill ID，与身份解耦。指标是不可变值，不是 React state。

Agent 是信任界面，不是 Skill 容器

03

席位定价

Starter: 3 位 Agent（免费/999元）。Professional: 5 位（2,999元）。Enterprise: 7 位全团队（8,999元）。更多 Agent = 更高产能。

自然升级路径

关键洞察：Agent 不是 Skill 的容器。Agent 是一个信任界面（Trust Interface），接收客户请求，选择合适的 Skill 执行，结果归因于 Agent。客户信任 Agent，Agent 调用 Skill，Skill 查询 KG。三层解耦。
04

L3 — 技能商店（Skill Marketplace）— 收入引擎

Skill 是原子化、可组合的能力插件。免费基础技能吸引用户，高级技能按使用量或订阅收费。OpenClaw 生态提供技能供应链，灵阙提供信任运行时。
类别免费层高级版企业版
基础记账凭证录入, 银行对账多币种, 合并报表自定义科目体系
税务申报增值税, 企业所得税加计扣除, 跨境税务转让定价, 税收协定
合规检查基础合规扫描行业专项, 风险预警自定义规则引擎
审计辅助异常检测抽样策略, 审计底稿持续审计

锁定机制

安装配置：每个 Skill 安装后需要针对客户业务做校准，切换意味着重新配置。

准确率积累：Skill 通过使用反馈持续改进，新竞品从零开始。

技能组合：复杂工作流串联多个 Skill，安装越多集成越深。

收入分成

自研 Skill：100% 归平台。

第三方 Skill（OpenClaw 生态）：平台 70% / 开发者 30%。

定价模型：按次使用 / 月订阅 / 一次性购买，三种灵活选择。

05

护城河分析

三道护城河，按复制难度递增排列。最强护城河不是技术（KG），而是时间积累的信任记录 — 竞品无法复制客户 6 个月的使用历史。
容易复制

UI 设计、页面布局、配色方案、基础 Skill（增值税申报）

难以复制

KG（514K 节点）：6-12 个月复制周期。Skill 生态：网络效应飞轮。但这是贬值资产，需持续注入新法规。

不可复制

信任记录：林税安服务腾讯 6 个月，准确率 98.2%，零事故。工作流锁定：10+ Skill 串联成定制流程。

07

风险与缓解

风险等级影响缓解措施
KuzuDB 存档状态 HIGH KG 基础设施无维护，安全漏洞 6 个月内评估 Neo4j CE / Apache AGE 迁移
HTTPS/HTTP 混合内容 MED 浏览器阻止 KG API 调用 Phase 1 实施 CF Worker 代理
1 人团队 vs 系统复杂度 HIGH B2 复杂度制动环主导，迭代减速 分拆后产品侧 ~12 页，显著降低认知负荷
OpenClaw 外部依赖 MED 平台政策变化导致 Skill 供应中断 核心 Skill 自研，OpenClaw 仅作补充来源
Agent 合规错误（B1 环） HIGH 客户被税务局处罚 → 口碑崩溃 Phase 3 人工监督 + 置信度阈值 + 强制人工确认
系统终结者风险：B1 合规制动环一旦激活，具有不对称破坏性。财税领域的错误不是"体验差"，是法律责任。Phase 3 必须设置置信度阈值（低于 80% 强制人工审核）和完整的审计日志。
