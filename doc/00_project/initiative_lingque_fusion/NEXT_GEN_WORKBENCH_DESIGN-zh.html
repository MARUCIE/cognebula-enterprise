下一代智能代账工作台
深度设计方案

基于 41 任务真实代账工作流 + OpenClaw 主动 Agent + 批量自动化引擎，将代账公司从手工作坊升级为日历驱动的智能生产线。
01

核心命题：代账公司是工厂，不是办公室

代账公司每月对 999 家企业执行相同的 41 道工序。挑战不是「做什么」（规则已知且确定性），而是「如何在零差错、零逾期的前提下批量执行」。这是工厂调度问题，不是知识工作问题。

五大设计决策

01

日历驱动

  • Agent 按日历日期自动激活，不等人类点击
  • 4 个时间窗口 = 4 个生产班次
  • 征期 15 号法定截止日 = 红色警报
02

管线并行

  • 企业 A 可在任务 5，企业 B 仍在任务 4
  • 逐企业流水线 + 批级同步点
  • 3-5x 吞吐量提升
03

信任车间主任

  • Agent = 具名数字员工，积累信任记录
  • 12 个月零差错 = 不可替代的护城河
  • 79 个 OpenClaw Skill 可独立升级
IMPLICATION

灵阙工作台的竞争壁垒不是 AI 技术本身（可复制），而是「7 个数字员工在你的 999 家客户数据上零差错运行 12 个月」后积累的信任记录（不可复制）。

05

依赖图引擎：两种依赖类型

关键洞察：依赖存在于「逐企业」和「批级」两个层面。混淆这两种类型会导致不必要的阻塞或过早执行。

逐企业依赖 (Per-Enterprise)

企业 X 的任务 B 依赖于企业 X 的任务 A 完成。

企业 001: 任务4 记账 --> 任务5 质检 --> 任务6 调整
企业 002: 任务4 记账 --> 任务5 质检

企业 001 可在任务 5 时，企业 002 仍在任务 4。这是管线并行——关键效率增益。

批级依赖 (Batch-Level)

任务 B 依赖于任务 A 对所有企业完成。

全部 999 企业完成 T15-T22 --> T23 漏报检查

批级依赖较少但关键，用于聚合操作（审计、报告、完整性检查）。整个系统中仅 2 处批级依赖点。

W1-W2 核心依赖链

T1 银行流水 T2 工资表 T3 报销单 --> T4 票据记账 --> T5 质检 --> T6 调整 --> T7 结账 --> T9 税项清册
T13 税金记账 --> T14 税金结账 --> T15-T22 8类申报(并行) --> T23 漏报检查(批级) --> T24 申报反馈(HITL) --> T25 税金统计
征期红线：任务 15-22（8 类税种申报）必须在 15 号前全部完成。系统在 13 号触发预警，14 号触发红色警报。任何未完成的申报自动升级为最高优先级。

人机协作协议：3 个刚性介入点

41 个任务中只有 3 个需要客户人工授权。其余 38 个任务可以完全自动化。

任务 24: 税种申报反馈

类型：客户同意扣款

流程：Agent 发送申报汇总 --> 客户回复「同意扣款」 --> Agent 执行划扣

超时：24h 提醒 / 48h 二次提醒 / 72h 升级总监

任务 28: 全盘账务反馈

类型：客户确认准确

流程：Agent 发送账务报告 --> 客户回复「确认无误」 --> 进入质检闭环

超时：同上

任务 29: 全盘异常调账

类型：客户书面授意 (法律要求)

流程：Agent 标记异常 --> 提供调账方案 --> 客户选择并签字 --> Agent 执行

超时：不自动跳过，标记「待客户决策」

通知通道优先级：微信服务号(主) --> 短信(备) --> 电话(总监升级)。客户 96h 无响应的非关键任务自动标记「客户未确认」并继续后续流程。

知识引擎：不可见但不可或缺

知识图谱（514K 节点 / 1.1M 边）是 Agent 准确率从 95% 提升到 99.5% 的核心——但客户永远不知道它存在。

没有 KG 的 Agent

准确率 ~95%

999 企业 x 5% 错误率 = 50 个错误/月

每个错误 = 客户投诉 + 潜在罚款 + 信任损失

有 KG 的 Agent

准确率 ~99.5%

999 企业 x 0.5% 错误率 = 5 个错误/月

可管理的异常量，每个都有 KG 辅助的根因分析

KG 查询热点 (按任务)

任务KG 查询用途
T5 质检LawOrRegulation --> TaxType验证凭证是否符合最新法规
T9 税项清册TaxType --> TaxRate获取当前企业类型的税率
T18 增值税TaxPolicy --> Industry行业特定增值税规则
T26 全盘质检ComplianceRule --> AccountType全面审计规则集
T31 风险报告Enterprise --> RiskFactor含历史数据的风险模型
IMPLICATION

竞品可以复制 UI 和 Agent 交互模式，但无法复制 514K 节点的专业知识图谱。这是 System A (CogNebula) 对 System B (灵阙) 的隐性战略价值——客户买的是「高准确率」，不是「知识图谱」。
