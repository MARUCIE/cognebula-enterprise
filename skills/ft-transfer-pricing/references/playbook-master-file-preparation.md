# 场景：主体文档准备

**政策依据**：国家税务总局公告2016年第42号第十四条；OECD BEPS行动计划13（三层文档结构）

## 第一步：确定是否需要准备主体文档

按42号公告第十三条，属于跨国企业集团且符合以下条件之一的企业应准备主体文档：
- 年度关联交易总额超过10亿元
- 已被要求准备本地文档的企业（关联交易达到第十五条标准）

主体文档由集团最终控股方或指定企业统一编制，各成员企业可共享。准备时限：次年12月31日前。可以使用集团总部编制的英文版主体文档，但税务机关要求时需提供中文翻译。

## 第二步：主体文档核心内容

按42号公告第十四条，主体文档应包含：

**（一）集团组织架构**：法律架构图（含各实体名称、注册地、持股比例）、经营架构图（重大交易各方的关系）、管理架构（关键管理决策的地点）

**（二）集团业务描述**：主要业务线描述、影响利润的重要交易和安排、主要产品/服务及前五大产品的收入占比、重要的集团内部劳务安排、主要市场的供应链描述、重要的商业重组/收购/处置

**（三）无形资产情况**：集团无形资产开发/持有/使用的整体战略、重要无形资产清单及法律所有权人、关联方之间重要的无形资产转让/许可安排及对价、集团转让定价政策中与无形资产相关的部分

**（四）融资情况**：集团融资安排描述（含关联方融资）、集团内部资金池或现金归集安排、集团内部借贷的定价政策

**（五）财务与税务状况**：集团合并财务报表、集团现行的单边/双边/多边预约定价安排和税收裁定清单

## 第三步：与本地文档的关系

主体文档（Master File）提供集团全局视角，本地文档（Local File）聚焦本地实体的具体关联交易。两份文档应保持一致——主体文档中的功能描述、无形资产归属、融资安排应与本地文档的功能风险分析相呼应。

如果集团总部提供的主体文档与中国本地实际不符（如集团描述本地公司为"有限风险分销商"但实际承担了重大市场风险），本地文档应如实反映，并说明差异。

## 第四步：实操要点与归档

主体文档编制通常由集团转让定价团队或外部顾问主导，中国子公司财税团队应配合提供本地相关信息。常见配合事项：本地公司的组织架构和人员信息、本地关联交易明细、本地无形资产使用情况、本地融资安排详情。

归档要求：与本地文档保存期限一致（10年）。每年更新——即使集团架构未变，财务数据和交易金额需更新。

关键输出：主体文档正文（含5大板块）+ 集团架构图 + 无形资产清单 + APA/裁定清单 + 更新记录。
