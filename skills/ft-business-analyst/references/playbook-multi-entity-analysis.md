# 场景：多主体架构分析

**政策依据**：《企业所得税法》第四十一条（关联交易）；《公司法》；各税收优惠政策

## 第一步：识别多主体架构模式

| 架构模式 | 典型结构 | 税务动机 |
|----------|----------|----------|
| 母子公司 | A公司100%持有B公司 | 风险隔离+子公司独立享受优惠 |
| 兄弟公司 | 老板分别持有A/B/C公司 | 分散收入/分别享受小微优惠 |
| 个人+企业混合 | 有限公司+个体户+个独 | 利用不同主体的税率差异 |
| 持股平台 | 有限合伙→运营公司 | 股权激励/投资收益税务安排 |
| 供应链拆分 | 贸易公司+服务公司 | 利用不同税率 |

## 第二步：架构合理性审查

**合法架构**：
- 每个主体有独立的经营场所、人员、业务
- 各主体间交易具有真实商业实质
- 关联交易价格符合独立交易原则
- 各主体独立核算、独立申报

**违规风险信号**：
- 拆分业务仅为让多个主体分别享受小微优惠（应纳税所得额≤300万）
- 各主体共用人员/场所/设备但分别申报
- 关联交易价格明显偏离市场价
- 主体间资金频繁往来且无合理业务支撑

**决策规则**：税务机关有权将形式上独立但实质为统一经营的多个主体合并计算，追缴税款。

## 第三步：架构税负对比分析

**分析模板**（以"兄弟公司"架构为例）：

| 比较项 | 单一公司 | 拆分为2家 |
|--------|----------|-----------|
| 总收入 | 500万 | A:250万 + B:250万 |
| 所得税税率 | 25%（超小微标准） | 各5%（均满足小微） |
| 所得税额 | 500万×25%=125万 | 250万×5%×2=25万 |
| 节税额 | - | 100万 |
| 合规风险 | 无 | 高（可能被合并计算） |

## 第四步：管理建议

- **架构设计时**：确保每个主体有真实独立的商业目的、经营场所、核心人员
- **日常运营**：各主体独立签订合同、独立收付款、独立开具发票
- **关联交易**：保留完整的定价依据和交易记录
- **年度审查**：每年评估架构的税务风险等级和优化空间
- **红线警示**：任何仅以节税为目的的架构设计都有被税务反避税调整的风险
