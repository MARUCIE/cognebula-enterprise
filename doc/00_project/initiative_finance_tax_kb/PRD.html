1. 项目目标

基于 CogNebula Enterprise 构建 SOTA 级中国财税知识操作系统:

  • 三层知识图谱: L1 法规中心 + L2 操作中心 + L3 合规中心 -- 107 张 KuzuDB 表, 43 种节点, 64 种边
  • 知识沉淀管线: 爬取 (20 信源) → 消化 (NER + 分类) → 管理 (Obsidian 知识库) → 发布 (CF Pages)
  • 现有 AI-Fleet 技能生态 (40+ 技能) 用于内容采集 -- 零自研爬虫代码
  • 中国专属本体: 18 税种, 42 项会计准则, 91 个科目, 7 个生命周期阶段, 6 种企业类型
  • 混合 Graph RAG + 12 个 MCP 工具 + OpenClaw 集成
  • 每日增量更新, 3 级变更检测
  • 全自动化: 日常运维零人工干预

设计原则

  1. 组合, 不构建: 编排 40+ 现有 AI-Fleet 技能, 不写自研爬虫
  2. 中国优先: 所有信源、Schema、NER 模式、专家角色均为中国税务体系设计
  3. 权威层级: P0 政府信源为基准真相; 媒体/社区仅作补充

2. 问题陈述

痛点

  1. 信息碎片化: 税收政策分散在 30+ 个政府网站 (国家税务总局、财政部、国务院、海关总署、央行...), 无统一机器可读 API
  2. 时间复杂性: 政策有生效/失效日期、修正案、属地覆盖 (国家 → 省 → 市)、优惠叠加 -- 人工跟踪极易出错
  3. AI Agent 上下文缺口: 中国财税领域不存在 Agent 可用 Cypher 查询的结构化知识图谱
  4. 合规滞后: 每日发布新法规; 18 税种 × 31 省份 = 规则组合爆炸

3. 项目范围

范围内

  • 通过现有 AI-Fleet 技能采集内容 (无自研爬虫):
    • news-aggregator-skill: 扩展 finance-tax profile + 8 个新信源抓取器
    • agent-reach: 多平台爬取 (政府门户、微信)
    • browser-automation (Midscene): 视觉驱动 JS 页面处理 (税务总局法规库)
    • multi-search-engine: 17 引擎发现 (百度、搜狗等)
    • deep-research: 7 阶段 GoT 合成, 用于复杂政策分析
    • defuddle / x-reader: 从任意 URL 提取干净文章
    • wechat-article-search: 微信公众号内容 (镜像方式)
    • scrapling-crawler: 反 Bot 绕过, 用于受保护站点
  • KuzuDB Schema: 13 种节点 + 15 种关系 (中国税务本体)
  • LanceDB 向量嵌入 (Gemini Embedding 2 Preview, 通过 bin/embed)
  • 3 级变更检测: 哈希 + 行级差异 + 语义相似度
  • OpenClaw MCP 集成: 5 个新财税查询工具
  • 专家团队: 2 个新角色 (秦税安 -- 税务, 顾财道 -- 财务)
  • 管线注册: 纳入 AI-Fleet pipeline-registry.json
  • 交付方式: Telegram 提醒 + CF Pages 仪表板
  • PDF/OCR: PyMuPDF + PaddleOCR, 用于政府公报
  • 编排脚本: scripts/finance-tax-crawl.sh (~100 行, 组合技能)

范围外 (v1)

  • 自研 Scrapy 爬虫 (使用现有技能生态)
  • 北大法宝 pkulaw.com (机构付费墙 + 刑法第 285 条法律风险)
  • 裁判文书网批量爬取 (v2 可选, 仅选择性抓取)
  • 省级税务局爬取 (与国家级高度重复; v2 可添加前 5 省)
  • 中国以外多管辖区 (v2)
  • NER 模型微调 (先用基于规则的正则)
  • 商业 SaaS 封装

时间模型

  • 双时态: valid_time (政策有效期) + transaction_time (知识库记录时间戳)
  • 版本链: TaxRateVersion 节点 + SUPERSEDES 边
  • 状态枚举: active | amended | repealed | pending | proposed
  • 冲突解决: 层级低者优先 (国发 > 财税 > 税总)
