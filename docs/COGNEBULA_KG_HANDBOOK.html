Section 01

什么是 CogNebula 知识图谱

CogNebula 是一个专注于中国业财税（业务-财务-税务）领域的知识图谱系统。它将散落在数百部法律法规、 数千条会计准则、上万条税率规则中的专业知识，编织成一张结构化的语义网络。在这张网络中，每一条税率 都能追溯到它的法律依据，每一笔业务活动都能找到对应的申报义务和合规要求——不再需要翻阅堆积如山的 红头文件，也不再需要凭经验猜测某条规则是否仍然有效。

从技术实现上看，CogNebula 底层基于 KuzuDB 图数据库，辅以 LanceDB 向量存储实现语义搜索能力。 图谱包含法律法规、会计准则、税率规则、合规要求、税收优惠、分类编码等多种类型的专业知识节点， 以及它们之间丰富的语义关系。所有数据均经过质量门禁审核后入库，确保数据的准确性和一致性。

架构上，CogNebula 采用三层设计：法规层（L1）存储法律文件原文与条款拆分， 业务层（L2）承载税率、科目、申报等日常运营知识，合规层（L3） 定义规则、风险指标与处罚标准。三层之间通过共享参考层（税种、分类、地区等）实现跨层语义遍历， 使得从一条法规条文出发，可以一路追溯到具体的税率适用、申报要求，直至合规风险。

当前规模：407,038 个节点、858,398 条边、边密度 2.1，覆盖 19 种税收类型（增值税、企业所得税、 个人所得税、消费税等全部主要税种）。质量评分 100/100，标题覆盖率 99.3%，边密度 3.977。 这是一个持续生长的知识体，每次法规更新、税率调整、政策变动，都会通过入库管线自动融入图谱。

Section 06

查询示例

1. 查询所有税种

获取系统中所有 19 种税种的 ID 和名称。

-- Cypher
MATCH (n:TaxType)
RETURN n.id, n.name
# curl
curl "http://100.75.77.112:8400/api/v1/nodes?type=TaxType&limit=19"

2. 增值税相关的法规

以增值税节点为起点，遍历所有关联节点（税率、法规条款等）。

-- Cypher
MATCH (t:TaxType {id: 'TT_VAT'})-[r]-(n)
RETURN t.name, type(r) AS rel, label(n) AS nodeType, n.id, n.name
LIMIT 50
# curl
curl "http://100.75.77.112:8400/api/v1/graph?table=TaxType&id_field=id&id_value=TT_VAT&depth=1"

3. 某条法规的所有条款

给定法规文件 ID，查出该文件下的全部条款。

-- Cypher
MATCH (c:LegalClause)-[:PART_OF]->(d:LegalDocument {id: 'LD_VAT_LAW_2024'})
RETURN c.clauseNumber, c.title, c.content
ORDER BY c.clauseNumber
# curl
curl "http://100.75.77.112:8400/api/v1/graph?table=LegalDocument&id_field=id&id_value=LD_VAT_LAW_2024&depth=1"

4. 小规模纳税人的税率

查找所有适用于小规模纳税人的税率规则。

-- Cypher
MATCH (e:TaxEntity {taxpayerStatus: '小规模纳税人'})<-[:APPLIES_TO_ENTITY]-(r:TaxRate)
RETURN r.name, r.value, r.calculationBasis, r.effectiveDate
ORDER BY r.name
# curl -- search API for semantic match
curl "http://100.75.77.112:8400/api/v1/search?q=小规模纳税人税率&limit=10"

5. 合规风险检查

查看业务活动关联的合规规则及对应处罚标准，用于风险预评估。

-- Cypher
MATCH (b:BusinessActivity)-[:GOVERNED_BY]->(c:ComplianceRuleV2)-[:PENALIZED_BY]->(p:Penalty)
RETURN b.name AS activity,
       c.name AS rule,
       c.category,
       p.name AS penalty,
       p.penaltyType,
       p.dailyRate
# curl -- RAG chat for natural language query
curl -X POST http://100.75.77.112:8400/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "哪些业务活动有违规处罚风险?"}'

入库门禁规则

  • title 字段长度 >= 2 个字符
  • content 字段长度 >= 20 个字符
  • 禁止原始 ID 泄露（如 UUID、数据库自增 ID）
  • 日期字段必须符合 ISO 8601 格式
  • 节点 ID 必须使用语义前缀（如 TT_ / LD_ / LC_ / CR_ 等）
