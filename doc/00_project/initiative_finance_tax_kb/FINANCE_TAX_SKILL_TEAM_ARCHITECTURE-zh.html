Phase 1 -- 代账核心 (13 Skills)

路由器

team-finance-tax
财税专家团路由器
意图分类 → 专家分发 → 协作编排 → 结果综合。识别问题类型（记账/申报/合规/咨询/工商），判断复杂度（单专家/多专家/圆桌），分发给对应 Skill。简单问题（"增值税税率是多少"）直接由路由器用内嵌知识回答，不调专家。

知识注入模式

Skill 只存决策规则和计算公式，不存原文法条。法规全文通过 kg-query MCP tool 按需从 KG 查询。行业深度通过 references/ 分行业子目录实现。
Skill 层级 references/ 放什么 为什么
路由器
team-*		不放知识。只放路由决策树和协作编排规则路由器的 token 预算应全给编排逻辑，不浪费在领域知识上
专家 Skill
ft-accountant 等		决策规则 + 计算公式 + 边界案例 + 行业子目录
references/industry-*.md		不放原文法条（太长、更新频繁）。放"已消化"的判定条件，如"小规模季度 ≤ 30 万免增值税"
工具 Skill
ft-regulation-lookup 等		MCP tool 调用说明 + 查询模板 + 返回值 schema教模型如何构造 KG 查询，而不是预存法规内容
