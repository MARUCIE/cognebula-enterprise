Three Key Findings

  1. 45% 爬虫仅抓标题。 22 个爬虫中有 10 个只抓取标题或摘要，完全缺乏正文内容。这意味着接近一半的信源管线产出的是"空壳节点" — 有实体名称，无实质知识，无法生成有效三元组。
  2. 18 个关键信源缺失。 财税领域的核心权威来源（全国人大法规库、总局政策库、中注协准则、法院裁判文书等）尚未接入。这直接制约了 ontology 的覆盖面和权威性。
  3. 密度悖论：边增长必须是节点增长的 8.7 倍。 要在 1M 节点规模下达到 density 6.0，需要 6M 条边。当前 860K 条边意味着缺口 5.14M。每新增 1 个节点，平均需要同时产出 8.7 条边 — 否则密度反而下降。
Verdict: REDESIGN

M3 的 L3 层必须从"节点生成引擎"转型为"边生成引擎"。盲目扩充信源只会加剧密度稀释和孤儿节点堆积。

4. Systems Dynamics Analysis (Meadows)

运用 Donella Meadows 的系统动力学框架，识别 CogNebula 知识图谱增长中的 4 个核心反馈回路。理解这些回路是制定正确战略的前提。

R1 Depth Enhancement REINFORCING — CURRENTLY INACTIVE

Deeper crawl → Better Q&A pairs → More edges → Higher density → Better queries → User feedback → Better crawl priorities → Deeper crawl

R2 Orphan Death Spiral REINFORCING — NEGATIVE

Fast injection → Edges lag behind → Orphan rate up → Quality gate cleanup → Need more injection to compensate → Even worse ratio

B1 Quality Gate Balance BALANCING

More nodes → Harder maintenance → Gate failures → Pipeline slowdown → Fewer nodes

B2 Density Dilution BALANCING

More nodes without edges → Density drops → Graph utility declines → Investment questioned → Fewer resources

The Core Math

Target density = 6.0 at 1M nodes
Required edges = 1,000,000 × 6.0 = 6,000,000
Current edges = 860,000
Edge gap = 6,000,000 − 860,000 = 5,140,000
New nodes needed = 1,000,000 − 408,000 = 592,000
Edges per new node = 5,140,000 / 592,000 = 8.68

Conclusion: Every new node entering the graph must carry an average of 8.7 edges. Pure node injection without edge generation is structurally self-defeating.

5. Revised Strategy — Edge-First M3

基于 Section 4 的系统动力学分析，M3 战略从传统的"先广度后深度"重新排序为"先深度后广度"，核心引擎从节点生产转向边生产。

L1 Depth (W1-7) L2 Breadth (W8-12) L3 Edge Engine (W13-15) L4 Expert Gate (W16)

L1 — Depth Enhancement (W1-7)

修复 Group B 爬虫的全文抓取能力。对每一个新节点自动运行 QA Generation 和 NER+Entity Linking，确保入库时自带 3-5 条边。激活反馈回路 R1。

  • fetch_chinaacc, fetch_chinatax_api 升级到 L3 全文
  • fetch_provincial 扩展到全部 31 省
  • 每条全文自动生成 Q&A 对 + 实体关系三元组
  • 验收标准：新节点平均边数 ≥ 3.0

L2 — Breadth Expansion (W8-12)

按 Tier 1-2 优先级接入新信源，但必须通过"准入协议"才能进入主图。

Admission Protocol

  1. New nodes enter staging graph (isolated namespace)
  2. Run NER + entity linking against main graph
  3. Minimum 3 edges found per node → promote to main graph
  4. Batch density < 3.0 → entire batch rejected, return to depth pipeline

L3 — Edge Engine (W13-15)

这是 M3 的核心创新。L3 不再生成新节点，而是专门在已有节点之间发现和创建关系。

  • Gemini Pro 扫描相邻节点对，识别 SUPERSEDES / CONFLICTS_WITH / APPLIES_TO 关系
  • 法规时间线：自动标注废止、修订、替代关系
  • 跨域桥接：税法 ↔ 会计准则 ↔ 执法案例 ↔ 裁判文书
  • 目标：单周产出 ≥ 500K 条新边

L4 — Expert Review & Final Gate (W16)

人工专家抽检 + 全图质量审计。确认 density ≥ 6.0，orphan rate < 1%，quality score 维持 100/100。
